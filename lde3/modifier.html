第三之書瀏覽器版修改器（v0.2）— 跨平台 / 純前端 / 無上傳。

支援檔案：lde3s.le3（主存檔，985,542 bytes）位於遊戲資料夾/usr/。
已知 header：byte 0-3 = "le3\0" magic / byte 4-31 = "LUNATIC DAWN - The 3rd Book -" + nulls / byte 32-63 = "Copyright (C) ARTDINK Corp. 2000" + nulls / byte 0x54+ = binary game data。

⚠️ 修改前務必備份：
1️⃣ 先把整個 usr/ 資料夾複製為 usr_backup/
2️⃣ 在備份上測試 + 載入遊戲驗證
3️⃣ 確認 OK 後再覆蓋原檔

本工具當前功能（v0.2）：
▸ 拖放 lde3s.le3 → header 驗證 + 結構區段識別
▸ Hex viewer（前 1024 bytes）+ 完整 hex 編輯
▸ ★ Big5 字串搜尋（找玩家姓名 / 城鎮 / 物品在 SAV 內位置）
▸ ★ 區段熱圖（每 16KB 統計 non-zero ratio + 變化區）
▸ ★ 跨版本 diff（上傳 .bak / .bk2 找 byte 差異 → 推測哪些 offset 對應「上一次存檔之後的變動」）
▸ 下載修改後 SAV

技術說明：lde3s.le3 是 MFC CArchive 序列化資料流（非 raw struct dump），跨檔差異率高，直接「固定 offset 修改」不可行。本工具改用 search-and-replace + diff 找變化區段方式輔助修改。

2026 結構分區發現（4 個玩家存檔 byte diff 取得）：
  • 0x00-0x6B (108 bytes)：固定 header — magic le3\\0 + ASCII 版權字串 + version u32=0x3d
  • 0x6C-0x18F (約 256 bytes)：半固定區 — 玩家可變只 24 bytes：
    • 0x72-0x7E: 13 bytes 玩家識別 / 屬性
    • 0x88-0x8E: 7 bytes 玩家屬性
    • 0xA6: 1 byte (推測種族 / 性別)
    • 0x16C-0x16E: 3 bytes (推測等級 / 年齡)
  • 0x190+ (約 1MB)：MFC CArchive object stream，每次 save 整體 re-serialize（99.7% byte 在跨存檔間都不同）
★ 24 bytes 識別 SOP（推薦做法）：
  1. 準備 USER0 backup：複製 lde3s.le3 → lde3s_backup.le3
  2. 記下「修改前狀態」：用本工具上傳，記錄 0x72-0x7E / 0x88-0x8E / 0xA6 / 0x16C-0x16E byte 值
  3. 進遊戲只改 1 個變數：例如「升 1 級」「改名」「改性別」其中一個（不要同時改多個！）
  4. 存檔後比對：上傳新 lde3s.le3，看 24 bytes 哪個變了 → 該 byte = 你修改的變數欄位
  5. 記下對照表：例如「0x72-0x73 = 等級」「0xA6 = 性別」
  6. 未來直接 offset-based 修改：知道 offset 後就能用本工具的 hex editor 直接改值，不用再進遊戲

⚠️ 0x190+ 動態流區段不適用 offset-based 修改（每次 save 整體重排）— 那邊只能用本工具的「Big5 字串搜尋 + diff」找特定字串再改。

所有解析在瀏覽器本地完成，不上傳任何資料。
