俠客遊 IV 存檔檢視器（v0.1）— 跨平台 / 純前端 / 不上傳 / 免安裝。

支援檔案：LDTSV0.dat ~ LDTSV9.dat（10 個存檔槽）及 LDTSVFIX.dat / LDTSVC.dat 等，位於遊戲資料夾/Usr/。

為什麼做這個：俠客遊 IV 的記憶體工具「神之眼 ld4navi.exe」是 2000 年針對日文版 ver.1.2.1.5 做的——它的存檔解析器寫死了日文版的欄位位置，讀繁體中文版存檔會因 layout 不符而出錯。這個檢視器是直接針對繁體中文版 LDTSV0.dat 做的，瀏覽器拖放即看，不需要安裝、不需要登錄檔、不需要任何外掛。

本工具當前功能（v0.1）：
▸ 拖放 LDTSV*.dat → magic 驗證 + header 結構解析
▸ 存檔標題（Big5 角色名）+ 存檔來源路徑解析
▸ ★ Big5 字串掃描（掃出檔案內所有可解碼的中文片段）
▸ ★ 區段熱圖（每區塊 entropy / 非零比例視覺化）
▸ 完整 hex 檢視（含已知區段標註）

技術說明：繁中版 LDTSV0.dat 的 0x00-0x6D 是可讀的 header（magic + 標題 + 來源路徑），0x70 之後是 Delphi/MFC 物件圖記憶體傾印（serialized object graph，含大量原始指標值）——沒有加密，但欄位語意需要進一步逆向才能對應到「等級 / 屬性 / 物品」。所以 v0.1 是檢視器（read-only 分析），不是數值編輯器。

所有解析在你的瀏覽器本機完成。檔案（含其中的存檔來源路徑）不會上傳到任何地方。

掃描整個檔案，列出所有可解碼為 Big5 的中文片段（≥ 2 字）。

⚠ 誠實提醒：0x70 之後是物件圖傾印，裡面隨機的位元組組合常常「剛好」是合法 Big5 編碼但解出來是亂碼。只有 header 區（0x20 標題）的字串是可靠的；掃描結果裡其餘多數是誤判，請對照 offset 自行判斷。

★ 區段熱圖

把檔案切成等分區塊，用顏色顯示每塊的 entropy（資訊密度）。亮綠 = 高密度物件資料；暗 = sparse / padding。點一格看該區段細節。

關於「神之眼」與這個檢視器

「神之眼 ld4navi.exe」是 2000 年由社群為日文版俠客遊 IV ver.1.2.1.5 製作的記憶體 / 存檔工具。它有兩個先天限制讓它讀不了繁體中文版：

  • 讀執行中遊戲：靠掃描遊戲行程的 heap、比對寫死的日文版記憶體簽章——繁中版記憶體配置不同，簽章對不上。
  • 讀存檔：解析器用寫死的日文版欄位位置 / 步距——繁中版 LDTSV0.dat 的 layout 不同，解析會爆界出錯。

這些是版本相容性問題，不是 bug——硬要神之眼吃繁中版，等於把它整個重寫。所以這個瀏覽器版檢視器是直接針對繁中版存檔做的：拖放即看、跨平台、零安裝、零外掛——對玩家來說才是真正「懶人」的方式。

v0.1 先做到 header 解析 + 結構分析。把 0x70+ 物件圖的欄位語意完整逆向（對應到等級 / 屬性 / 物品）是後續工作。
