Lunatic Dawn IV (2001) · 次元之門與連線系統 · 台灣第三波代理

俠客遊 IV（Lunatic Dawn IV）是 2001 年由台灣第三波代理發行的系列作品，加入了次元之門與 IP 連線功能。本站收錄 toni 的連線設定方法、鞠躬練功法，以及 Morrowind 作的 LunaticConsortia 輔助工具三個歷史版本。

⚠️ 第三波官方伺服器於 2007 年前後關閉，現代要連線需透過 IP 直連方式。詳見下方「連線方法」攻略。

[ 光碟結構分析（2026 逆向）]

2026 年 toni 將正版遊戲實體光碟拷貝到電腦後，本站對 LD4 完整光碟結構做了二進位分析：

主程式

  • LD4.exe — 2,561,024 bytes（約 2.5 MB），編譯日期 2001-05-25，PE32 格式（Windows 32-bit i386 GUI）
  • 跟 LD3（2000）僅相差 6 個月發行；採同樣 MFC 框架但加入連線功能
  • 遊戲資料總大小約 334 MB（比 LD3 多 60 MB，多在 Character 跟 Effect）

★ Web/ 模板系統（LD4 獨有）

LD4 內建 Web/Models/ 4 種主題的 HTML 模板，玩家世界檔上傳到伺服器後會生成對應 HTML 頁面：

  • standard 標準主題：title 寫「LD IV WORLD INFORMATION」
  • colors / games / users：不同視覺主題
  • 每個主題含 5 個頁面：index / world / top / menu / player

從 Web/Models/standard/world.html 可看到玩家世界資訊頁的欄位設計：

  • HOME WORLD IP ADDRESS（核心 — 別人連你世界的進入點）
  • 世界名稱 / 公開狀態 / 開放狀況 / 容許人數
  • 訪客履歷：近 4 位訪客名單（World 名稱 + 玩家名稱 + 訪問時間 + 退出時間）

⚠️ 第三波官方伺服器於 2007 年前後關閉。但 IP 直連功能仍可用 — 詳見上方▸ 連線方法。

遊戲資料分類（Data/）

LD4 沿用 LD3 的 9 大類資料結構（系列共通設計）：

  • Map 107 MB — 地圖資料（最大）
  • Monster 82 MB — 怪物 sprite（增加大量新怪物）
  • Character 43 MB — 角色 sprite（比 LD3 多 19 MB）
  • Effect 8.1 MB — 戰鬥特效（比 LD3 多 3.7 MB）
  • Weapon / Shield 約 4 MB — 武器 / 盾牌資料
  • Campaign / Range / MSG — 劇本 / 範圍 / 訊息字典
  • usr/（LD4 新增）— 使用者存檔資料夾

⚠️ 怪物 / 角色 sprite 是創作性表現（屬版權保護內容），本站不公開。

.rsrc 內含 UI 字串（2026 提取）

從 LD4.exe .rsrc resource section 提取出 448 個唯一中文 UI 字串（UTF-16LE 編碼，比 LD3 多 148 個）。LD4 獨有功能字串：

  • 網頁開啟功能：「開啟網頁」/「詳細資料」/「遊戲情報」
  • 世界主題系統：standard / colors / games / users 4 種主題對應 Web/Models/ 模板

★ LD4 加密分析（2026 known plaintext attack）

2026 年用 known plaintext attack 對 LD4 加密 BIN 取得部分突破：

  • 已知：LD3 MonsterPara.BIN 是明文 (\\0LD3 magic + 12 bytes 0 padding)
  • 推測：LD4 MonsterPara.BIN 對應原檔開頭應為 \\0LD4 (差 byte 3：0x33→0x34)
  • 解出：用 \\0LD4 + 0 padding 反推前 24 byte XOR key：
    97 6e e2 ce bc bb 8a d5 00 55 3e 93 36 ee 32 3e 6a 87 66 d7 9e 20 9a 70
  • 驗證：解出 byte 0-3 = 00 4c 44 34 = \\0LD4，前 24 bytes 完全對應 ✓
  • 限制：byte 24+ 不是 cyclic XOR — cipher 切到 stream / state-based 算法。完整破解需要反組譯 LD4.exe 找 cipher init function。

→ 這是「**已加密但不安全**」的設計：用 zero padding 作為 known plaintext 立刻洩漏前 24 bytes XOR keystream。Artdink 顯然不是專業 cryptographer。

★ Artdink 加密策略演進（2026 跨檔對照發現）

從 LUNA2 (1996) 到第三之書 (2002)，Artdink 對遊戲資料的加密策略逐步加強：

作品 加密狀態
LUNA2 (1996)ENC 4-byte XOR（簡單，已破解）
LD3 (2000)完全明文（36 個 .BIN 全 \\0LD3 magic）
LD4 (2001)選擇性加密：40 個明文 + 4 個加密（ITEM_GUARD / ITEM_NORMAL / ITEM_WEAPON / MonsterPara.BIN，magic 0x9a 開頭）
第三之書 (2002).dat 檔全 Huffman 壓縮 + 加密

LD4 把最關鍵的 4 個資料檔加密（怪物參數 + 3 種道具表），其他世界 / 建築 / 地形仍明文。Artdink 顯然意識到「玩家會破解什麼」就先把那 4 種鎖起來。

res/ 完整檔案統計（44 BIN + 366 BMH + 33 TIL）

LD4 比 LD3 多了 8 個獨有 .BIN（如 Bld_Farm.BIN — 農場系統，Area_Dungeon_E.BIN — 地下城東區）。BMH 數量翻倍（187 → 366）跟 TIL 增多反映 LD4 美術資源大幅擴充。

資料安全

LD4 的網頁系統設計上，玩家世界的 IP address 直接寫進 HTML給其他玩家看到。在 2001 年是進步設計（玩家可以直接連線進入別人世界），但今天看是「IP 公開暴露」的設計。第三波官方伺服器於 2007 年前後關閉後，玩家只能用 LAN 內 IP 直連。
