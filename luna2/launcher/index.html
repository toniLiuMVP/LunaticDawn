[ ABOUT ]

俠客遊 II 是 1996 年的 DOS 遊戲。原本要在 macOS / Windows 11 / Linux 上跑起來，得自己裝 DOSBox、改 config.sys、 凹出 LH + UMB 把 DOS 載到高位記憶體、解決 640 KB 常規記憶體 不夠的問題，還要手動設音效卡 IRQ/DMA——每一關都能勸退 90% 的人。

這個啟動器把那些 1996 年的地獄一次處理好——下載 3 個檔案、 雙擊，就可以跑。跨 macOS / Windows / Linux。 音效使用 GM MIDI（General MIDI）高品質音樂模式。

※ 請先至 Steam 商店購買正版俠客遊 II 支持原作。 本站不提供遊戲本體下載，請自備 LUNA2.EXE 及原版資料檔。
※ 不想自己裝 DOSBox-X？試試 ▸ Luna2_EZ 懶人包——解壓即玩，內建引擎。

[ STEP 1 · 安裝 DOSBOX-X ]

DOSBox-X 是 DOSBox 的進階分支，對 1996 年繁中遊戲的相容性最好——支援 Big5 顯示、完整的 EMS/UMB/HIMEM 模擬、SVGA 模式，而且對 MIDPAK/DIGPAK 音效 TSR 驅動的相容性比 DOSBox Staging 更好，能正常使用 GM MIDI 音樂。

macOS

brew install --cask dosbox-x-app

沒有 Homebrew 可以到 dosbox-x.com 下載 .dmg 手動安裝。

Windows

winget install DOSBox-X.DOSBox-X

或到 dosbox-x.com 下載 .exe 安裝檔。

Linux

# Debian / Ubuntu
sudo apt install dosbox-x

# Arch
sudo pacman -S dosbox-x

# 或從 GitHub Release 下載 AppImage

※ 需要 DOSBox-X 0.84 或以上。舊的 DOSBox（非 X 版）也能跑，但需要改設定。

[ STEP 2 · 下載啟動器 ]

下載 dosbox-x.conf（必備）+ 你 OS 對應的啟動器 （Mac 下載 .command，Windows 下載 .bat，Linux 下載 .sh）。

▼ dosbox-x.conf ▼ Mac_Play_Luna2.command (macOS) ▼ Win_Play_Luna2.bat (Windows) ▼ Linux_Play_Luna2.sh (Linux)

※ 瀏覽器可能會把 .command / .bat 當成危險檔案。放心，原始碼都在 GitHub Repo 可以審查，每一行都看得見。

[ STEP 3 · 放進遊戲資料夾並執行 ]

3.1 把下載的 3 個檔案（dosbox-x.conf、 Mac_Play_Luna2.command 或 Win_Play_Luna2.bat） 全部放在和 LUNA2.EXE 同一個資料夾。

⚠ 重要：遊戲資料夾的完整路徑不能包含中文或特殊字元，否則 DOSBox 可能無法正確掛載。
✓ 正確範例：D:\Games\Luna2\、/Users/toni/Games/Luna2/
✗ 錯誤範例：D:\遊戲\俠客遊II\、/Users/toni/下載/俠客遊/
3.2 · macOS 使用者 第一次執行時右鍵點 Mac_Play_Luna2.command → 選「打開」→ 彈出警告 → 選「打開」。

或者終端機跑一次： 
chmod +x Mac_Play_Luna2.command
之後雙擊即可。
3.2 · Windows 使用者 直接雙擊 Win_Play_Luna2.bat。 Windows Defender 可能會警告「已防止無法辨識的應用程式啟動」， 點「其他資訊」→「仍要執行」。
3.2 · Linux 使用者 終端機先給執行權限： 
chmod +x Linux_Play_Luna2.sh
./Linux_Play_Luna2.sh
或者在檔案管理器右鍵 →「屬性」→ 勾選「允許作為程式執行」，之後雙擊即可。
3.3 DOSBox-X 視窗跳出 → 看到 綠色開場白 → 進入俠客遊 II → 選擇 USER 存檔槽 → 開始遊戲。

[ STEP 4 · 音效設定（推薦 GM MIDI）]

俠客遊 II 支援多種音效模式。本啟動器推薦使用 GM MIDI（General MIDI）——音質最好的模式， 搭配 DOSBox-X 內建的 MIDI 合成器可以呈現完整的背景音樂與音效。

設定 GM MIDI 音樂（範例 4 · try4-gm）

GM MIDI 需要兩個步驟：設定遊戲音效模式 + 載入 TSR 驅動。

4.1 · 設定遊戲音效模式 在 DOSBox-X 裡執行 SNDSET.EXE， 選擇 GM (General MIDI) 音樂模式。 設定完成後會產生 SNDSET.CFG。

如果你的遊戲資料夾已有 SNDSET.CFG 且音效正常，可以跳過此步。
4.2 · 載入 TSR 驅動程式 GM MIDI 需要在啟動遊戲前載入兩個 Miles Sound System 的 TSR 常駐程式。 在 DOSBox-X 的 DOS 提示字元下依序執行： 
LH MIDPAK.COM
LH M\SB16.COM
LUNA2.EXE
LH = Load High（載入到高位記憶體，節省 640K 常規記憶體）。 MIDPAK.COM 是 MIDI 音樂驅動， M\SB16.COM 是 Sound Blaster 16 音效驅動。

也可以把這三行加進 dosbox-x.conf 的 [autoexec] 區段，讓每次啟動自動載入： 
[autoexec]
@echo off
cls
echo  Lunatic Dawn II / 俠客遊 II
echo.
LH MIDPAK.COM
LH M\SB16.COM
啟動腳本會在 [autoexec] 後面自動補上 mount + LUNA2.EXE 指令。

如果 GM 沒有聲音？退回 Adlib

少數情況下 MIDPAK/SB16 TSR 可能和你的 DOSBox 版本不相容 （特別是 DOSBox Staging 不支援這些 TSR）。 如果載入 TSR 後仍然沒有音樂：

  1. 重新執行 SNDSET.EXE，改選 Adlib 模式
  2. 不載入 MIDPAK / SB16 TSR，直接執行 LUNA2.EXE
  3. Adlib 使用 DOSBox 內建 FM 合成，不需要額外驅動，穩定度最高

※ Adlib 的音質是 FM 合成（8-bit 電子風），GM MIDI 的音質是樂器取樣（更接近真實樂器）。 兩種都是完整的背景音樂，差別在音色。

[ 常見問題與排除 ]

Q1. 為什麼要用 DOSBox-X 而不是 DOSBox？

DOSBox-X 相容性比原版強很多——支援中文 Big5 顯示、完整的 EMS/UMB/HIMEM 模擬、 SVGA 模式、更精細的 CPU 週期控制、以及 dos = high,umb 這種模擬 MS-DOS 6.22 config.sys 的能力。 最重要的是，DOSBox-X 對 MIDPAK/DIGPAK TSR 驅動的相容性最好， 能正常使用 GM MIDI 音樂。

Q2. 遊戲跑太快 / 太慢？

按 Ctrl + F11 減速，Ctrl + F12 加速。 預設是 30000 cycles（約 486/Pentium 75 等級），對俠客遊 II 來說剛好。

Q3. 沒有音效 / 音樂？

依序檢查：

  1. 確認 SNDSET.CFG 存在於遊戲資料夾。 沒有的話執行 SNDSET.EXE 設定音效。
  2. 如果使用 GM MIDI 模式，確認已在 LUNA2.EXE 之前載入 LH MIDPAK.COM 和 LH M\SB16.COM（見 STEP 4）。
  3. 如果 TSR 載入後仍無聲，改用 Adlib 模式 （不需 TSR，DOSBox 內建 FM 合成）。
  4. 檢查 dosbox-x.conf 裡 nosound = false（預設就是 false）。

Q4. 存檔在哪裡？

存檔是 USER0 到 USER9 這 10 個資料夾（對應遊戲裡的 10 個存檔槽）。 注意 USER 編號 = 遊戲內槽號 - 1 （USER0 = 存檔 1，USER9 = 存檔 10）。

想修改存檔？試試看本站的 ▸ 瀏覽器版修改器——直接拖 LUNACHAR.SAV 進去就能改。

Q5. 找不到 LUNA2.EXE？

確認遊戲資料夾裡面有 LUNA2.EXE（主程式，大約 610 KB）。 如果是剛從磁碟片安裝，執行過 INSTALL.EXE 應該就會自動放好所有檔案。

Q6. macOS 說「未識別的開發者」？

這是 macOS Gatekeeper 的保護。解法：

chmod +x Mac_Play_Luna2.command
xattr -d com.apple.quarantine Mac_Play_Luna2.command

或者到「系統設定 → 隱私權與安全性」最下面允許執行。

Q7. 路徑含中文導致 DOSBox 無法掛載？

DOSBox-X 的 mount 指令對含中文字元的路徑支援不完整。 將遊戲資料夾搬到純英文路徑即可解決。 啟動腳本已內建中文路徑偵測，會在啟動前提醒你。

啟動腳本做了什麼？

  1. 切換到腳本自己所在的資料夾（所以放哪都能跑）
  2. 檢查路徑是否包含中文字元（有則警告）
  3. 檢查 DOSBox-X 是否存在（常見路徑 + PATH）
  4. 檢查 LUNA2.EXE 和 dosbox-x.conf 是否都在
  5. 執行 dosbox-x -conf dosbox-x.conf -c "mount c ." -c "c:" -c "LUNA2.EXE"

關鍵是 -c 參數——DOSBox-X 會把這些指令附加到 [autoexec] 區段後面執行。這樣 dosbox-x.conf 完全不用硬編路徑，任何人下載都能直接用。
