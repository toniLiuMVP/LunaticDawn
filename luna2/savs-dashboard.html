USERN 存檔結構總覽

屬性修改器 · 物品欄編輯 · 商店查看 · 回主站
俠客遊 II 玩家存檔在 USERN/（N=0~9，10 個 slots），每 slot 含 13 個 SAV 檔共約 740 KB。 本頁總覽全 13 SAV 結構研究進度（截至 2026-05-02 2026）+ drag-drop hex viewer。

已可編輯：LUNACHAR.SAV（屬性修改器）+ CHRITEM.SAV（物品欄編輯器 v0.2）。
已可查看：SHOPITEM.SAV（商店查看器）。
結構釐清：LUNAHEAD / KOBUN / OYABUN / HIST / SCENDATA / TOWNDATA（2026 binary survey）。
待逆向：TOWNITEM（92 KB） / VDD / VDDKEY（各 200 KB） / DUNGDATA（map data）。

LUNAHEAD.SAV 結構釐清

size: 256 bytes
內容：玩家存檔表頭概要
byte 0-2：00 47 09（u16 = 18176, 1801 — 可能是日期 / 等級）
byte 5-12：Big5 中文字串（玩家姓名）
byte 13-17：u32 = 87 + u16 = 48（等級 / age？）
byte 18+：0xFF padding

KOBUN.SAV 結構釐清

size: 542 bytes（不整除任何因子）
內容：子分（小弟）資料 — 玩家招募 NPC 後加入此檔
byte 0-13：u16 序列 0/0/1/13/13/16/32（小弟編號 + 屬性 + 等級）
非零比例：68.3%
對應：同行 NPC 招募流程

OYABUN.SAV 結構釐清

size: 258 bytes（不整除任何因子）
內容：親分（老大）資料 — 玩家當頭目時的隊伍狀態
byte 0-7：u32 = 1, 2（玩家親分編號 + 隊伍人數）
byte 12-25：u16 序列 971/7/12/10/30/7/10/3（屬性 / 等級概要）
非零比例：60.9%

HIST.SAV 結構釐清

size: 16,384 bytes (16 KB)
內容：玩家歷史紀錄 / 事件 log
byte 0-7：u16 序列 0/971/7/12（事件類型 + 日期）
byte 8+：大量 0xFF padding
結構：可能是「玩家事件紀錄表」每事件 N bytes

SCENDATA.SAV 結構釐清

size: 6,144 bytes (6 KB)
內容：場景資料 — 玩家進入過的場景紀錄
非零比例：11.7%（多場景未訪問）
byte 0-7：01 0c 00 06 ff ff 05 ff（u16 = 3073, 1536, 65535, 65285）
byte 8-15：49 00 ff ff 68 42 00 00（u16 = 73, 65535, 17000, 0）
因子：16/32/48/64/96/128 bytes/entry

TOWNDATA.SAV 結構釐清

size: 8,192 bytes (8 KB)
內容：城鎮資料 — 玩家在每城鎮的記錄
非零比例：77.4%（多城已訪問）
byte 0-7：01 00 bc da bc 77（u16 = 1, 55996, 30652）
因子：16/32/64/128/256/512 bytes/entry

TOWNITEM.SAV 待逆向

size: 92,160 bytes (90 KB)
內容：城鎮物品 — 玩家在城鎮 store 的物品堆積
因子：可能 96 × 960 bytes 或 192 × 480 bytes

VDD.SAV 待逆向

size: 204,800 bytes (200 KB)
內容：影像資料 — 推測為玩家 sprite cache / VDD pages
注意：VDD 可能跟 BCHRKP runtime 填入相關（2026發現 BCHRKP 全 0 是 template）

VDDKEY.SAV 待逆向

size: 204,800 bytes (200 KB)
內容：影像 key — VDD.SAV 的索引 / decryption key
關係：跟 VDD.SAV 同 size，疑似 paired 結構

DUNGDATA.SAV map data

size: 30,720 bytes (30 KB)
內容：玩家生成迷宮 layout（不是 metadata）
來源：DUNGDATA.MST template 全 0x00 → 遊戲安裝時複製到 USERN，玩家用過後填入 map
非零比例：91%
逆向波次：2026釐清 vs metadata 區分
