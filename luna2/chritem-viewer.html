CHRITEM.SAV 物品欄查看器

← 屬性修改器 · 回主站
俠客遊 II 玩家物品欄存於 USER0/CHRITEM.SAV（122880 bytes）。 本工具支援 查看 + 編輯（v0.2 2026加入 edit 模式），可解析 160 個角色（玩家 + NPC 同行）的物品欄。

⚠️ 修改前務必備份：本工具的 byte 0-7（item_id + value2）已 binary 逆向確認， 但 byte 8-47（modifier / 強化值）未實機驗證。修改後若載入遊戲出錯可能損壞角色。 建議流程：
1️⃣ 將 USER0/CHRITEM.SAV 先複製為 USER1/CHRITEM.SAV（DOSBox 內可選 USER1 = 存檔 2）
2️⃣ 在 USER1 測試修改 + 載入遊戲驗證
3️⃣ 確認 OK 後再覆蓋 USER0


結構（2026-05-02 2026逆向）：160 角色 × 16 物品 slots × 48 bytes/slot。 每 slot：byte 0-3 = item_id（對應 ITEMNAME.ENC 的 1-456 物品）；byte 4-7 = value2（推測為價格）； byte 8-47 = 強化值 + 屬性 modifier + 額外資料。

byte 8-47 結構性分類（推測）：
▸ byte 8-15（modifier ID 區）：8 bytes — 多為 0xFF 表「無 modifier」；含值時是「裝備類型 + 強化等級 + 屬性」組合（範例 02 1a 5d 70 01 ff 02 00）
▸ byte 16-23（屬性數值區）：8 bytes — 推測對應「7 系魔法上限」+ STM 加成（範例 0d 12 0f 14 0e 10 ff ff）
▸ byte 24-27（marker 區）：4 bytes — 多為 00 00 ff ff 或屬性類別標記
▸ byte 28-35（屬性 modifier 區）：8 bytes — 玩家屬性加成值（力/智/靈/敏/直/魅/體/精，範例 0a 04 0a 00 00 00 00 ff）
▸ byte 36-47（padding 區）：12 bytes 全 0 — 跨 10 USER 樣本 25,600 entries 統計確認 byte 37-47 純 padding (byte 17/19 也是)

2026 統計分析發現：
▸ byte 8 = main modifier type ID (8 種：0xff = 無 modifier / 其他 7 種類型)
▸ byte 9 / byte 16 = 高 diversity 數值欄位（推測為強化等級 / 耐久度 / 多餘屬性 strength）
▸ byte 26 = secondary modifier type (8 種)
▸ byte 28-35 = 8 個屬性 modifier values（對應 8 屬性 力/智/靈/敏/直/魅/體/精）

※ 個別 modifier ID 的精確效果對照仍需在 DOSBox 內實機驗證（sample 角色 + 變動單一 byte 觀察）。本 viewer 提供 read 模式（v0.1）+ edit 模式（v0.2）。

使用方式：拖放或選擇 USER0/CHRITEM.SAV 檔案 → 左側選角色 → 右側看物品欄。 所有解析在瀏覽器本地完成，不上傳任何資料。
