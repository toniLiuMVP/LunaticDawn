[ ABOUT ]

Luna2_EZ 是俠客遊 II 的「懶人包」—— 把 DOSBox Staging 引擎、啟動腳本、遊戲設定檔全部打包好， 你只需要放入自己的正版遊戲檔案，雙擊就能玩。 不用自己裝 DOSBox、不用改設定、不用處理音效驅動。

跨 macOS（含 macOS 26 Tahoe）/ Windows / Linux。

※ 請先至 Steam 商店購買正版俠客遊 II 支持原作。 本站不提供遊戲本體下載，請自備 LUNA2.EXE 及原版資料檔。
※ 想要自己安裝 DOSBox-X + GM 高音質音樂？請參考 ▸ 自己裝版本（DOSBox-X）。

[ 常見問題與排除 ]

macOS：「已損毀，無法打開」

macOS 26 (Tahoe) 對未經 App Store 發布的 App 檢查更嚴格。 如果內建 DOSBox Staging 被判定為「已損毀」，啟動器會自動切換到系統安裝版。

解法：用 Homebrew 安裝完整簽章的 DOSBox Staging：

brew install --cask dosbox-staging-app

安裝後重新雙擊 Mac_Play_Luna2.command 即可。

macOS：「未預期的結束」(Abort trap: 6)

macOS 26 的 Terminal.app 安全限制，直接執行 .app/Contents/MacOS/dosbox binary 會觸發 C++ 例外（AutoMountDrive）導致閃退。

懶人包的啟動器已經用 open -W -a 透過 macOS LaunchServices 啟動，繞過這個限制。如果你仍然遇到此問題，請確認使用的是最新版啟動腳本。

macOS：第一次執行被阻擋

macOS Gatekeeper 可能會阻擋 .command 腳本。

方法 A：右鍵點選 → 「打開」→ 確認打開。

方法 B：終端機執行：

chmod +x Mac_Play_Luna2.command
xattr -d com.apple.quarantine Mac_Play_Luna2.command

或到「系統設定 → 隱私權與安全性」最下面允許執行。

Windows：SmartScreen 警告

Windows Defender SmartScreen 可能會顯示「已防止無法辨識的應用程式啟動」。 點「其他資訊」→「仍要執行」。

沒有音樂 / 音效

懶人包使用 Adlib/OPL 音效模式，DOSBox 內建 FM 合成器， 理論上開箱就有背景音樂和音效。

如果沒有聲音，確認 game/SNDSET.CFG 存在且設定正確。 可以在 DOSBox 內執行 SNDSET.EXE 重新選擇 Adlib 模式。

路徑含中文導致無法啟動

DOSBox 的 mount 指令對含中文字元的路徑支援不完整， 可能出現「Unable to mount」錯誤。

解法：將整個 Luna2_EZ 資料夾搬到純英文路徑。 啟動器已內建中文路徑偵測，會在啟動前提醒你。

存檔在哪裡？

存檔位於 game/USER0 到 game/USER9（對應遊戲內 10 個存檔槽）。 注意 USER 編號 = 遊戲內槽號 - 1 （USER0 = 存檔 1，USER9 = 存檔 10）。

想修改存檔？試試 ▸ 瀏覽器版存檔修改器——拖入 LUNACHAR.SAV 就能改。

[ 技術說明 ]

為什麼用 Adlib 而不是 GM MIDI？

俠客遊 II 支援 General MIDI（GM）音樂，音質比 Adlib FM 合成好。 但 GM 模式需要先載入兩個 1996 年的 TSR 常駐驅動程式 （MIDPAK.COM + M\SB16.COM）， 這些驅動是 Miles Sound System 的產物， 和 DOSBox Staging 不完全相容—— 實測在 DOSBox Staging 下載入 TSR 後完全沒有音樂音效。

Adlib/OPL 使用 DOSBox 內建的 FM 合成器， 不需要任何額外驅動，穩定度最高。對懶人包來說「開箱即有音樂」比音質更重要。

※ 想要 GM 音質？請使用 ▸ 自己裝版本（DOSBox-X）， DOSBox-X 對 MIDPAK TSR 相容性較好。

macOS 26 (Tahoe) 的特殊處理

macOS 26 對 DOSBox Staging 有三層安全限制，懶人包啟動器已全部處理：

  1. com.apple.provenance 延伸屬性 —— 從網路下載的 .app 會被標記，且 macOS 26 的 SIP 保護讓這個屬性 無法用 xattr -cr 移除。如果內建引擎簽章損毀， 啟動器自動偵測並切換到系統安裝版。
  2. Terminal.app 安全限制 —— 直接在終端機執行 .app/Contents/MacOS/dosbox 會觸發 AutoMountDrive 例外而閃退。 啟動器使用 open -W -a 透過 LaunchServices 繞過。
  3. 崩潰恢復對話框 —— DOSBox 若異常退出，macOS 會彈出「是否重新開啟」對話框干擾下次啟動。 啟動器寫入 ApplePersistenceIgnoreState 防止此問題。
