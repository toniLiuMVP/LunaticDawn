  • ✅ 跨平台 — macOS(含 macOS 26 Tahoe)/ Windows / Linux 三平台啟動腳本
  • ✅ 解壓即玩 — 不用自己裝 DOSBox、不用改設定、不用處理音效驅動

[ ⚠️ 有音樂音效版「跳回 DOS / EMS Init Error」歷史問題 ]

▌ 真因確認:EMS 記憶體不足

2011-12-26 PTT Old-Games 板玩家 Draculalu 回報:

「在我準備離開神聖吉札帝國前打算去學一下物理系法術奧義,在走出迷宮時跳回 DOS 並顯示 EMS init error。延伸記憶體設定為 16M 應該可以正常執行才對...」

對應 2004 年青衫主攻略(本站 [ 俠客遊 II 主攻略 ] 內第 97-100 行)早就寫了:

「是要有 EMS 但又不能太多,若超過 16 MB 會出現 EMS Init Error,因此要設定 MS-DOS 模式的 EMS 記憶體,給個 4 MB 便夠了。此外,主記憶體的需求量也很高,若出現 Out Of Memory 便表示主記憶體不夠,最好設成無音效及無音樂,以降低主記憶體的需求,如此大約有 600 KB 便能玩了。」

▌ 為什麼有音樂音效會跳、無音樂音效不會跳

俠客遊 II 1996 年寫死了 EMS / conventional memory 邊界檢查:

  1. 遊戲對 DOS conventional memory(640 KB)需求極高 — 載入遊戲本體後接近吃完
  2. 載入音樂音效 driver(MIDPAK / DIGPAK / SB16.COM)需要 40-100 KB conventional memory + EMS
  3. 場景切換時(如離開迷宮)需要重新 mmap EMS 區塊 → 加上音效 driver 佔的記憶體 → 剩餘不夠 → 跳回 DOS
  4. 無音效版:沒載 MIDPAK / DIGPAK,記憶體寬鬆 → 不會 EMS init error

這就是過去 20+ 年 toni 版 Luna2 流傳「無音樂音效版」的真實原因 — 是 1996 年 DOS 遊戲記憶體配置的硬限制。

▌ 已知觸發場景

  • 學物理系法術奧義 → 走出迷宮(Draculalu 2011 案例)
  • 其他可能觸發點:長時間遊玩後場景切換、進入多個地下城、學多種高階法術後

▌ ✅ 2026-05-13 已主動預防(懶人包預設 memsize=4)

本懶人包已預設 memsize=4(對應 1996 俠客遊 II 設計目標配置,青衫 2004 攻略明示「給 4 MB 便夠了」),100% 避開遊戲內部 16-bit 整數溢位的 EMS init error bug。下載最新版懶人包就有此預防。

memsize=4 vs memsize=16 對玩家的差異:畫面、速度、音樂音效品質、操作體驗 — 完全一樣。唯一差別:memsize=4 100% 避開 1996 EMS 整數溢位 bug,memsize=16 會觸發。詳細技術說明 → GitHub commits。

▌ 萬一仍遇跳出 → 切回無音樂音效版(原始 toni 版做法)

編輯遊戲資料夾內 SNDSET.CFG 第 1 個 byte 設為 00(無音效),或執行附帶的還原無聲腳本。和 toni 過去 20+ 年流傳的版本一樣穩定。

▌ 仍遇跳出 → 寄信告訴 toni

不同場景觸發可能需要不同解法。寄信時請描述:

  • 跳出前場景(城鎮名 / 地下城 / 戰鬥)
  • 跳出前動作(學什麼法術 / 進什麼迷宮 / 對話)
  • 是否看到 EMS init error 或其他錯誤訊息
  • 遊玩時間(剛開始 / 已玩數小時)

▸ 來信回報跳出場景

玩家越多回報,toni 越能定位是哪些場景觸發 EMS 邊界。30 年前的 DOS 記憶體限制問題,讓我們用 2026 年的工具解決它。

[ 常見問題與排除 ]

macOS：「已損毀，無法打開」

macOS 26 (Tahoe) 對未經 App Store 發布的 App 檢查更嚴格。 如果內建 DOSBox Staging 被判定為「已損毀」，啟動器會自動切換到系統安裝版。

解法：用 Homebrew 安裝完整簽章的 DOSBox Staging：

brew install --cask dosbox-staging-app

安裝後重新雙擊 Mac_Play_Luna2.command 即可。

macOS：「未預期的結束」(Abort trap: 6)

macOS 26 的 Terminal.app 安全限制，直接執行 .app/Contents/MacOS/dosbox binary 會觸發 C++ 例外（AutoMountDrive）導致閃退。

懶人包的啟動器已經用 open -W -a 透過 macOS LaunchServices 啟動，繞過這個限制。如果你仍然遇到此問題，請確認使用的是最新版啟動腳本。

macOS：第一次執行被阻擋（一鍵解決）

macOS 26+ 對「網路下載的可執行檔」預設加 com.apple.quarantine 屬性，Gatekeeper 會靜默拒絕雙擊。懶人包裡所有 .command + DOSBox Staging.app 都會中。

★ 推薦做法（跑一次永久搞定）：

  1. 右鍵點懶人包根目錄的 _Mac首次執行_先點我.command （檔名底線開頭，Finder 排序在最上面）→ 選「打開」
  2. 跳出「來自身份不明的開發者」警告 → 再按一次「打開」
  3. 看到「✓ 已清除 macOS quarantine」訊息
  4. 之後直接雙擊 Mac_Play_Luna2.command 啟動遊戲

這個破冰腳本只需跑一次。它會清掉整個懶人包資料夾的 quarantine， 包括引擎、所有 .command、DOSBox Staging.app。

進階使用者也可以用終端機一鍵清整個資料夾：

xattr -dr com.apple.quarantine ~/Downloads/Luna2_EZ
chmod +x ~/Downloads/Luna2_EZ/*.command

Windows：SmartScreen 警告

Windows Defender SmartScreen 可能會顯示「已防止無法辨識的應用程式啟動」。 點「其他資訊」→「仍要執行」。

沒有音樂 / 音效

懶人包使用 Adlib/OPL 音效模式，DOSBox 內建 FM 合成器， 理論上開箱就有背景音樂和音效。

如果沒有聲音，確認 game/SNDSET.CFG 存在且設定正確。 可以在 DOSBox 內執行 SNDSET.EXE 重新選擇 Adlib 模式。

路徑含中文導致無法啟動

DOSBox 的 mount 指令對含中文字元的路徑支援不完整， 可能出現「Unable to mount」錯誤。

解法：將整個 Luna2_EZ 資料夾搬到純英文路徑。 啟動器已內建中文路徑偵測，會在啟動前提醒你。

存檔在哪裡？

存檔位於 game/USER0 到 game/USER9（對應遊戲內 10 個存檔槽）。 注意 USER 編號 = 遊戲內槽號 - 1 （USER0 = 存檔 1，USER9 = 存檔 10）。

想修改存檔？試試 ▸ 瀏覽器版存檔修改器——拖入 LUNACHAR.SAV 就能改。

[ 技術說明 ]

為什麼用 Adlib 而不是 GM MIDI？

俠客遊 II 支援 General MIDI（GM）音樂，音質比 Adlib FM 合成好。 但 GM 模式需要先載入兩個 1996 年的 TSR 常駐驅動程式 （MIDPAK.COM + M\SB16.COM）， 這些驅動是 Miles Sound System 的產物， 和 DOSBox Staging 不完全相容—— 實測在 DOSBox Staging 下載入 TSR 後完全沒有音樂音效。

Adlib/OPL 使用 DOSBox 內建的 FM 合成器， 不需要任何額外驅動，穩定度最高。對懶人包來說「開箱即有音樂」比音質更重要。

※ 想要 GM 音質？請使用 ▸ 自己裝版本（DOSBox-X）， DOSBox-X 對 MIDPAK TSR 相容性較好。

macOS 26 (Tahoe) 的特殊處理

macOS 26 對 DOSBox Staging 有三層安全限制，懶人包啟動器已全部處理：

  1. com.apple.provenance 延伸屬性 —— 從網路下載的 .app 會被標記，且 macOS 26 的 SIP 保護讓這個屬性 無法用 xattr -cr 移除。如果內建引擎簽章損毀， 啟動器自動偵測並切換到系統安裝版。
  2. Terminal.app 安全限制 —— 直接在終端機執行 .app/Contents/MacOS/dosbox 會觸發 AutoMountDrive 例外而閃退。 啟動器使用 open -W -a 透過 LaunchServices 繞過。
  3. 崩潰恢復對話框 —— DOSBox 若異常退出，macOS 會彈出「是否重新開啟」對話框干擾下次啟動。 啟動器寫入 ApplePersistenceIgnoreState 防止此問題。
