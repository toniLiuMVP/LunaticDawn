吟遊詩人的傳說 / 立即遊玩 EzPlay

[ 這是什麼 ]

EzPlay 補丁包專門解決「老遊戲在現代電腦跑不動」的各種問題，讓你用自己手上的正版遊戲，在現代 Windows / Mac 順利遊玩。

它不含遊戲本體，你要先有自己的正版遊戲（Steam 版或當年的原版光碟 / 磁碟片）。 把 EzPlay 解壓後覆蓋到遊戲本體根目錄，再執行裡面那一個啟動檔，所有問題一次解決。

⚖ 正版聲明：EzPlay 只包含「讓遊戲能在現代系統執行」的修正補丁，不散布任何遊戲本體 / 美術 / 劇情 / 音樂。 請先透過 Steam 俠客遊系列 或你手上的正版光碟取得遊戲本體，支持原作 Artdink。

[ 通用 3 步驟 ]

  1. 準備遊戲本體：把你的正版遊戲安裝好（Steam 安裝目錄，或原版光碟安裝後的資料夾）。
  2. 解壓覆蓋：把對應的 EzPlay .7z 解壓，把裡面所有檔案複製貼到遊戲本體的根目錄（也就是遊戲主程式 .exe 所在的資料夾），遇到「是否覆蓋」選全部覆蓋。
  3. 執行那一個啟動檔：雙擊對應你系統的啟動檔（Windows 雙擊 .bat / Mac 雙擊 .command），即可開始遊玩。
⚠ 請執行啟動檔，不是遊戲主程式：Mac 一定要用 .command 啟動檔（它會呼叫 Wine 並處理首次執行）；Windows 雙擊 .bat 啟動檔最保險（會自動切到正確目錄）。

※ Mac 首次執行若被系統擋下，先雙擊 _Mac首次執行_先點我.command 一次（解除 Gatekeeper 隔離），之後就正常。Mac 需先安裝 Wine。

啟動檔幫你做了什麼？為什麼要用它？

老遊戲不是雙擊主程式就能在現代系統跑，啟動檔幫你把執行環境準備好（透明、可檢視，純文字檔可以自己打開看）：

  • Mac：① 自動找到你電腦裡的 Wine（老 Windows 遊戲要靠 Wine 才能在 Mac 上執行，啟動檔會依序找 Homebrew Wine / Wine Crossover / CrossOver）→ ② 解除 macOS 對下載檔的安全隔離（Gatekeeper，否則 .command 會被系統擋住點不開）→ ③ 切換到遊戲所在資料夾 → ④ 用 Wine 啟動遊戲。
  • Windows：先切換到遊戲所在資料夾再啟動主程式，這樣遊戲才找得到同資料夾的必要元件（直接雙擊主程式有時工作目錄不對，會出問題）。

啟動檔是純文字檔，你可以用記事本／文字編輯器打開檢視它做了哪些事，完全透明。

💡 排除經驗（這款最曲折，值得記錄）

難題：遊戲進得去，但一點「工具列視窗」就閃退。真兇竟是「想把視窗放大」的嘗試。

俠客遊 IV 是 2001 年的多子視窗程式（工具列、地圖、角色狀態各是獨立小視窗）。我們一度想把視窗放大（老遊戲在大螢幕顯得很小），做法是加一個元件去攔截系統的「視窗繪圖」指令，結果它把子視窗的繪圖座標搞亂了，一點工具列就崩潰。

解法：把「放大視窗」的攔截全部移除，只留遊戲本身必要的最小元件，工具列就恢復正常、不再閃退。想要大視窗請改用 Windows 內建顯示縮放（設定 → 系統 → 顯示器 → 縮放），讓系統去放大，不要在程式裡硬攔截。

順帶取捨：原版的背景音樂原本在 CD 音軌，沒有光碟就沒有音樂；曾試過用 mp3 自動播放補回，但有相容問題（Mac 尤其無解），最終取捨為無背景音樂、換取 100% 穩定不閃退。教訓：對老程式注入元件時，能不碰的（視窗 / 繪圖 / DPI 縮放）就別碰，越精簡越穩。

💡 排除經驗

老實說，第三之書是這三款裡最順手的一款 😄。它是 2002 年的 Windows 原生遊戲（系列裡唯一不是 DOS 的一作），在現代 Windows 上幾乎是放進去就能玩，沒有前兩作那些字體、視窗、音樂的老毛病。

主要工夫在 Mac：第三之書當年只發行 Windows 版，Mac 沒有原生版本，得靠 Wine 才能執行。我們把這件事交給啟動檔自動處理（找 Wine、解除安全隔離、設好執行環境，流程見上方「啟動檔幫你做了什麼？」），讓 Mac 玩家也能一鍵遊玩。所以這款的「困難」不在遊戲本身，而在怎麼讓老 Windows 遊戲在 Mac 上順順地跑起來，這正是 EzPlay 啟動檔幫你解決的事。
